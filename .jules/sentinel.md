## 2025-03-05 - SQL Injection in Kubernetes Init Containers
**Vulnerability:** Shell scripts in initContainers executing `psql` queries used direct variable interpolation (`$DB_POSTGRESDB_USER`) within strings (e.g., `"CREATE USER $DB_POSTGRESDB_USER WITH PASSWORD '$DB_POSTGRESDB_PASSWORD';"`). This is vulnerable to SQL injection if environment variables are ever populated with malicious input containing SQL meta-characters.
**Learning:** Even though the input might currently be sourced securely from a Kubernetes Secret, treating secret values as safe against injection vulnerabilities is a bad practice. Variables intended as identifiers or string literals in SQL must be sanitized or properly parameterized. The `psql` command provides safe interpolation natively.
**Prevention:** When writing `psql` queries in shell scripts (e.g., init containers), use `psql -v param="value"` parameterization. Then use `:'param'` to escape string literals and `:"param"` to escape identifiers.
